How it works

A clear flow from school setup to daily use—teachers, students, and parents each get the tools they need.

1. School setup

A school admin creates the school and submits it for approval. Once approved by Orion13, the school can add classes, streams, and teachers. Class teachers are assigned to specific classes.

2. Class lists & student IDs

Class teachers create class lists (e.g. Primary 5 East). Each student gets a unique Student ID (e.g. LH-2025-ABC-123456) and an access code. Teachers can import names from a document or add students one by one.

3. Joining the school

Students: Sign up with email or Google, then join using their Student ID and access code. No approval needed—instant access to assignments, notes, timetable, and results.

Teachers: Submit a join request; the school admin or class teacher approves.

Parents: Link children by entering each child's Student ID and access code. View results and activity for all linked children.

4. Daily use

Teachers post assignments and notices, share notes (documents and videos), build mark lists, enter marks, and publish results. Everyone gets push notifications for new content.

Students see assignments, open notes, check timetables, and view published results. Notifications keep them on track.

Parents see assignments and results for linked children. Results are accessed via Student ID and access code if not yet linked.

5. Results & report cards

When a class teacher publishes a mark list, students and linked parents see the results. Parents can also enter a Student ID and access code to view a child's results before linking.

Unique Student IDs — Format: LH-YEAR-XXX-NNNNNN. Each student has one ID per school. When a teacher is reassigned to a class, the new teacher continues from where the previous one left off.

Notifications — Admins get notified when a new school is created or verified; school admins get notified of join requests and when results are published; students and parents get notified of new assignments and notices.
